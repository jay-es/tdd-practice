- 第 1 の法則: テストを書くまでは本番コードを書いてはならない。本番コードがないためにテストは失敗する。
- 第 2 の法則: 失敗するテストやコンパイルできないテストを必要以上に書いてはならない。失敗を解決するには本番コードを書く。
- 第 3 の法則: 失敗しているテストを解決する本番コードを必要以上に書いてはならない。テストがパスしたら、追加のテストを書く。

- ルール 2: 失敗させる。パスさせる。クリーンにする。
